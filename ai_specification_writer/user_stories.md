# User Stories

### User Story 1
**As a student, I want to create an account so that I can access my personalized study dashboard.**

**Acceptance Criteria:**
- Users can register using an email address and password.
- The system verifies the email before account activation.
- The user is redirected to the dashboard after successful login.

**Priority:** MVP

---

### User Story 2
**As a student, I want to create and manage my study schedule so that I can organize my learning effectively.**

**Acceptance Criteria:**
- Users can add, edit, and delete study sessions.
- The schedule is automatically saved.
- Upcoming sessions are displayed on the dashboard.

**Priority:** MVP

---

### User Story 3
**As a student, I want to upload my class notes so that I can keep all study materials in one place.**

**Acceptance Criteria:**
- Users can upload PDF, DOCX, and TXT files.
- Uploaded files are stored securely.
- Files can be viewed or downloaded later.

**Priority:** MVP

---

### User Story 4
**As a student, I want AI to summarize my notes so that I can review important concepts quickly.**

**Acceptance Criteria:**
- Users can request a summary for uploaded notes.
- The system generates a concise summary.
- The summary is displayed within a reasonable time.

**Priority:** High

---

### User Story 5
**As a student, I want AI to generate quizzes from my study materials so that I can test my knowledge.**

**Acceptance Criteria:**
- Users can generate quizzes from uploaded notes.
- Quiz questions are based on the uploaded content.
- The system displays the user's score after completion.

**Priority:** High

---

### User Story 6
**As a student, I want to receive reminders about upcoming study sessions so that I do not miss important tasks.**

**Acceptance Criteria:**
- Users can enable or disable reminders.
- Notifications are sent before scheduled sessions.
- Reminder times can be customized.

**Priority:** High

---

### User Story 7
**As a student, I want to join study groups so that I can collaborate with classmates.**

**Acceptance Criteria:**
- Users can create or join study groups.
- Group members can exchange messages.
- Members can share study materials.

**Priority:** Medium

---

### User Story 8
**As a teacher, I want to upload learning materials so that students can access course resources online.**

**Acceptance Criteria:**
- Teachers can upload learning resources.
- Students can access shared materials.
- Files remain available until removed by the teacher.

**Priority:** Medium

---

### User Story 9
**As a teacher, I want to create assignments so that I can evaluate student learning.**

**Acceptance Criteria:**
- Teachers can create and publish assignments.
- Students can submit their work online.
- Teachers can review submitted assignments.

**Priority:** Medium

---

### User Story 10
**As an educational administrator, I want to view learning analytics so that I can monitor student engagement and performance.**

**Acceptance Criteria:**
- The system displays student activity statistics.
- Reports can be filtered by class or course.
- Analytics can be exported for reporting.

**Priority:** Low
