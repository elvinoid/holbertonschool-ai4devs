# Productivity Comparison

The following table compares the benchmark tasks completed manually versus with GitHub Copilot assistance.

| Metric | Without AI | With AI | Difference |
|--------|-----------:|--------:|-----------:|
| Average completion time (min) | 21 | 8 | -62% |
| Total completion time (min) | 63 | 25 | -60% |
| Lines of code (LOC) written | 165 | 110 | -33% |
| Bugs or test failures encountered | 4 | 1 | -75% |

## Analysis

Using GitHub Copilot significantly improved development productivity. The average completion time decreased from **21 minutes** per task to **8 minutes**, resulting in approximately **60% time savings**.

AI assistance also reduced the amount of code that had to be written manually. Copilot generated much of the boilerplate code, allowing the developer to focus on reviewing and refining the implementation instead of writing repetitive code.

The number of bugs and test failures was also reduced. AI-generated suggestions followed common Express.js patterns and best practices, producing fewer implementation errors. Manual review remained necessary to verify correctness, improve readability, and ensure the generated code met the project requirements.

Overall, AI-assisted development increased productivity while maintaining code quality, although developer oversight was still essential before accepting generated code.
