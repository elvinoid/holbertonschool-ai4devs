# Prompt Use Cases

## Code Quality

### Refactoring
- **Goal**: Improve readability, maintainability, and efficiency.
- **Input**: Source code or function in any programming language.
- **Output**: Refactored code with an explanation of the improvements.

### Style Enforcement
- **Goal**: Apply consistent naming conventions and code formatting.
- **Input**: Code block or project files.
- **Output**: Reformatted code following language style guidelines.

### Performance Optimization
- **Goal**: Reduce execution time or memory usage.
- **Input**: Existing implementation.
- **Output**: Optimized code with performance recommendations.

---

## Debugging

### Error Diagnosis
- **Goal**: Identify the cause of compilation or runtime errors.
- **Input**: Source code and error message.
- **Output**: Explanation of the issue and suggested fix.

### Logic Analysis
- **Goal**: Find logical mistakes that produce incorrect results.
- **Input**: Function or algorithm.
- **Output**: Root cause analysis and corrected implementation.

### API Troubleshooting
- **Goal**: Resolve incorrect usage of libraries or frameworks.
- **Input**: Source code using an external API.
- **Output**: Correct API usage with implementation guidance.

---

## Documentation

### Code Documentation
- **Goal**: Generate clear comments and documentation.
- **Input**: Source code or functions.
- **Output**: Inline comments and documentation blocks.

### README Generation
- **Goal**: Create project documentation.
- **Input**: Project structure and description.
- **Output**: A complete README with installation and usage instructions.

### Technical Summarization
- **Goal**: Explain complex code in simple language.
- **Input**: Source file or module.
- **Output**: Concise technical summary.

---

## Testing

### Unit Test Generation
- **Goal**: Create unit tests for application code.
- **Input**: Source code or function.
- **Output**: Unit test cases with expected results.

### Test Case Design
- **Goal**: Generate normal, boundary, and edge-case inputs.
- **Input**: Function specification or implementation.
- **Output**: Comprehensive test scenarios.

### Test Failure Analysis
- **Goal**: Explain why automated tests fail.
- **Input**: Test output and source code.
- **Output**: Root cause analysis and recommended fixes.
